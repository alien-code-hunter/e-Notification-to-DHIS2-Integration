# DHIS2 User Role Assignment Checklist for EN - Death Notification

## 1. Identify User Groups
- [ ] Program Officers (data review, reporting)
- [ ] Data Entry Clerks (data entry, editing)
- [ ] Supervisors/Admins (verification, rejection, user management)
- [ ] IT/Technical Support (system maintenance, troubleshooting)

## 2. Assign Roles in DHIS2
- [ ] Grant access to the "EN - Death Notification" program for all relevant user groups
- [ ] Assign permissions for:
    - [ ] Event capture (data entry)
    - [ ] Event editing
    - [ ] Event verification/rejection (for supervisors/admins)
    - [ ] Viewing and exporting data (for program officers)
    - [ ] API access (for technical users)
- [ ] Restrict access to sensitive data as needed (PII protection)

## 3. Test User Access
- [ ] Log in as each user type and confirm correct access to the program
- [ ] Verify that only authorized users can verify/reject events
- [ ] Ensure program officers can view and report on data
- [ ] Confirm technical users can access API endpoints if required

## 4. Document and Communicate
- [ ] Document all assigned roles and permissions
- [ ] Communicate assignments to all users
- [ ] Provide support contacts for access issues

---

Update this checklist as roles or requirements change.
